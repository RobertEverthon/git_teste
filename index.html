Utilizamos o comando 'git init' no terminal para ativa-lo. Utilizamos o comando 'clear' para limpar o terminal. Utilizamos o comando 'git status' para ver os comandos a serem versionados(ver o que está acontecendo no projeto). Utilizamos o comando 'git add .' para versionar a pasta inteira. Utilizamos o comando 'git add .\(nome do arquivo)' para versionar apenas um arquivo. Após isso, utilizamos o comando 'git commit -a -m ' para fazer o commit ( -a significa "todos os arquivos" e -m significa "adicionar mensagem, para caso haja necessidade de explicar alguma alteração ou atualização, a mensagem tem que estar entre aspas"). Utilizamos o comando 'git remote add origin "endereço do repositório"' para conectar ao repositorio a pasta. Utilizamos o comando "git branch -M main" para dizer que ele é o branch principal da aplicação. Utilizamos o comando "git push -u origin main" para enviar o codigo atual para o repositório. novo: "git checkout -b" vai criar um novo branch separado do original (o comando 'checkout' muda o branch que esta sendo versionado no terminal ex:'git checkout main' vai mudar para o repositorio 'main') Utilizamos o comando "git push --set-upstream origin (nome do novo branch)" para criar um ponto no branch. "git branch" mostra o branch que você está. 'git merge teste' comando que vai mesclar o branch 'teste' no branch original 'main'. 'git pull' vai tranferir os arquivos que foram adicionados e atualizados no repositorio para o local(ou seja, aqui) "git clone" vai colonar o repositorio.
